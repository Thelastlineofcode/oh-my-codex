# Research Skill

Deep research and information gathering.

## When to Use
- Evaluating technology options
- Understanding new APIs/libraries
- Investigating best practices
- Comparing solutions
- Learning new concepts

## When NOT to Use
- When answer is already known
- Simple documentation lookups
- Time-critical tasks

## Research Process

### 1. Define Question
```
What exactly do we need to know?
- Specific question
- Success criteria
- Scope boundaries
```

### 2. Gather Sources
```
Primary sources:
- Official documentation
- GitHub repos
- Academic papers
- RFC specifications

Secondary sources:
- Blog posts
- Stack Overflow
- Community forums
- Video tutorials
```

### 3. Analyze & Compare
```
For each option:
- Pros and cons
- Use cases
- Limitations
- Community support
- Maintenance status
```

### 4. Synthesize
```
- Summary of findings
- Recommendation
- Supporting evidence
- Caveats/risks
```

## Usage

```
research: compare React state management solutions

research: best practices for API rate limiting

research: evaluate database options for time-series data
```

## Output Guidelines

1. **Be specific** - Cite sources, version numbers
2. **Be objective** - Present tradeoffs fairly
3. **Be actionable** - End with clear recommendation
4. **Be current** - Note when info might be outdated
